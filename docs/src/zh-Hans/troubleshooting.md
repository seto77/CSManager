# 故障排查

汇总常见问题及处理方法。

- **无法启动 / 找不到 .NET 运行时** — CSManager 的运行需要 **.NET Desktop Runtime 8** 或更高版本。若尚未安装，请从 Microsoft 的发布页面获取。
- **COD 读取失败** — `读取 COD 数据库（含 ### 个晶体）` 在首次会从互联网下载大容量文件（超过 800 MB）。请检查网络连接和磁盘剩余空间，并稍后重试。
- **无法打开数据库文件** — 请确认 `*.cdb3` 文件是否已损坏、被移动，或以其他版本保存。若直接使用 AMCSD / COD，可通过 `读取 AMCSD 数据库（含 ### 个晶体）` / `读取 COD 数据库（含 ### 个晶体）` 重新读取。
- **大型数据库下运行缓慢** — 在 COD 这类大规模数据库中，检索和读取可能较为耗时。请先按所需条件筛选后再操作。
- **无法与 PDIndexer / ReciPro 协同** — 请确认协同目标软件是否正在运行。选中时的自动转送经由剪贴板进行（参见 [与其他软件的协同](6-interoperation.md)）。

此处未列出的问题，请向 GitHub 的 [Issue](https://github.com/seto77/CSManager/issues) 反馈。
