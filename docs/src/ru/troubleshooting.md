# Устранение неполадок

Ниже приведены распространённые проблемы и способы их решения.

- **Не запускается / не найдена среда .NET** — для работы CSManager требуется **.NET Desktop Runtime 8** или выше. Если она не установлена, скачайте её со страницы распространения Microsoft.
- **Не удаётся загрузить COD** — при первом использовании `Загрузить базу COD (кристаллов: ###)` из интернета скачивается большой файл (более 800 МБ). Проверьте подключение к сети и наличие свободного места на диске, после чего повторите попытку через некоторое время.
- **Не удаётся открыть файл базы данных** — проверьте, не повреждён ли файл `*.cdb3`, не перемещён ли он и не сохранён ли в другой версии. Если вы используете AMCSD / COD в неизменном виде, можно загрузить их заново через `Загрузить базу AMCSD (кристаллов: ###)` / `Загрузить базу COD (кристаллов: ###)`.
- **Медленная работа с большой базой данных** — для крупных баз данных, таких как COD, поиск и загрузка могут занимать время. Сначала отфильтруйте данные по нужным условиям, а затем выполняйте операции.
- **Не работает взаимодействие с PDIndexer / ReciPro** — проверьте, запущена ли целевая программа. Автоматическая передача при выборе выполняется через буфер обмена (см. [Взаимодействие с другими программами](6-interoperation.md)).

О проблемах, не описанных здесь, сообщайте через [Issue](https://github.com/seto77/CSManager/issues) на GitHub.
