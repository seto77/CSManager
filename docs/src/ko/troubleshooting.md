# 문제 해결

자주 발생하는 문제와 대처 방법을 정리합니다.

- **실행할 수 없음 / .NET 런타임을 찾을 수 없음** — CSManager의 동작에는 **.NET Desktop Runtime 8** 이상이 필요합니다. 설치되어 있지 않은 경우에는 Microsoft의 배포 페이지에서 받으십시오.
- **COD 읽기에 실패함** — `COD 데이터베이스 읽기 (### 개 결정 포함)`는 최초에 인터넷에서 대용량 파일(800 MB 초과)을 다운로드합니다. 네트워크 연결과 빈 디스크 용량을 확인하고, 시간을 두고 다시 시도해 주십시오.
- **데이터베이스 파일을 열 수 없음** — `*.cdb3` 파일이 손상, 이동, 또는 다른 버전으로 저장되지 않았는지 확인해 주십시오. AMCSD / COD를 그대로 사용하는 경우에는 `AMCSD 데이터베이스 읽기 (### 개 결정 포함)` / `COD 데이터베이스 읽기 (### 개 결정 포함)`에서 다시 읽을 수 있습니다.
- **큰 데이터베이스에서 동작이 느림** — COD와 같은 대규모 데이터베이스에서는 검색이나 읽기에 시간이 걸릴 수 있습니다. 필요한 조건으로 좁힌 후에 조작해 주십시오.
- **PDIndexer / ReciPro와 연동할 수 없음** — 연동 대상 소프트웨어가 실행되어 있는지 확인해 주십시오. 선택 시 자동 전송은 클립보드를 통해 이루어집니다([다른 소프트웨어와의 연동](6-interoperation.md) 참조).

여기에 기재되지 않은 문제는 GitHub의 [Issue](https://github.com/seto77/CSManager/issues)로 보고해 주십시오.
