# Carregamento do banco de dados

O carregamento e o salvamento do banco de dados são feitos pelo menu `Arquivo`.

## Bancos de dados padrão

- `Ler banco de dados AMCSD (### cristais)` — carrega o banco de dados AMCSD fornecido pela Mineralogical Society of America (MSA). Contém cerca de 20 mil estruturas cristalinas e está incluído no arquivo de instalação, dispensando download separado. O número atual de registros é exibido no menu. Se a opção "Ler banco de dados AMCSD na próxima inicialização" estiver ativada em `Opções`, ele será carregado automaticamente na inicialização.
- `Ler banco de dados COD (### cristais)` — carrega o banco de dados de cristais fornecido pelo Crystallography Open Database (COD). Contém mais de 500 mil estruturas cristalinas e, por ter um tamanho de arquivo grande (mais de 800 MB), não está incluído. No primeiro carregamento, ele é baixado automaticamente da internet e, nas vezes seguintes, o arquivo já baixado é carregado.

## Bancos de dados personalizados

- `Ler banco de dados` — abre qualquer arquivo de banco de dados do CSManager (`*.cdb3`).
- `Salvar banco de dados` — salva o banco de dados atual no formato `*.cdb3`.
- `Limpar banco de dados` — limpa os dados já carregados. O arquivo de banco de dados em si não é excluído.

Ao usar o AMCSD ou o COD diretamente, raramente haverá necessidade de usar `Ler banco de dados` / `Salvar banco de dados`.

## Uso e citação do banco de dados

Ao publicar artigos acadêmicos ou similares utilizando os dados de estrutura cristalina contidos no AMCSD, seja on-line ou off-line, cite sempre o seguinte artigo.

> Downs, R.T. and Hall-Wallace, M. (2003) The American Mineralogist Crystal Structure Database. *American Mineralogist* 88, 247-250.
