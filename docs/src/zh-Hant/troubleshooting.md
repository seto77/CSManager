# 疑難排解

以下整理常見問題與處理方法。

- **無法啟動 / 找不到 .NET 執行階段** — CSManager 的執行需要 **.NET Desktop Runtime 8** 以上。若尚未安裝，請從 Microsoft 的散布頁面取得。
- **COD 讀取失敗** — `讀取 COD 資料庫（含 ### 個晶體）` 在首次會從網際網路下載大容量檔案（超過 800 MB）。請確認網路連線與可用磁碟空間，並隔一段時間後再試。
- **無法開啟資料庫檔案** — 請確認 `*.cdb3` 檔案是否毀損、被移動，或以其他版本儲存。若直接使用 AMCSD / COD，可從 `讀取 AMCSD 資料庫（含 ### 個晶體）` / `讀取 COD 資料庫（含 ### 個晶體）` 重新讀取。
- **大型資料庫下動作緩慢** — 在 COD 這類大規模資料庫中，搜尋與讀取可能較為耗時。請先以所需條件篩選後再操作。
- **無法與 PDIndexer / ReciPro 連動** — 請確認連動對象的軟體是否已啟動。選取時的自動傳送是經由剪貼簿進行（請參閱 [與其他軟體的連動](6-interoperation.md)）。

此處未記載的問題，請回報至 GitHub 的 [Issue](https://github.com/seto77/CSManager/issues)。
